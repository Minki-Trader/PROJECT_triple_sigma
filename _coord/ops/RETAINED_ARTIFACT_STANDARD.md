# Retained Artifact Standard

Scope:
- Applies to retained tester evidence folders under:
  - `_coord/artifacts/step19_*`
  - `_coord/artifacts/step20_*`

Required files per retained artifact directory:
- `README.md`
- `manifest.json`
- `trade_log.csv`
- `exec_state.ini`
- `tester_log_tail.txt`
- all `bar_log_YYYYMMDD.csv` files generated by the run

Required README fields:
- artifact title
- source preset
- source summary
- validation class:
  - control
  - live-pass regression
  - live-opposite probe
  - recovery probe
  - reject synthetic negative-path
- trigger source:
  - actual-path
  - synthetic trigger
  - synthetic reject
  - recovery reload probe

Required manifest fields:
- artifact name
- created_at
- preset
- summary
- validation_class
- trigger_source
- synthetic flags
- file hashes
- trade-log counts
- duplicate `(trade_id,event_type)` count
- same-timestamp `EXIT -> ENTRY` count
- `MODIFY` count

Single-run cleanliness rule:
- `tester_log_tail.txt` must contain only the log bytes appended by the target
  run.
- mixed-run contamination is a packaging failure.
- If the original retained tail is empty or mixed, it may be rebuilt from the
  same-run tester agent log, but:
  - the artifact `README.md` must say so explicitly
  - the rebuilt tail must still represent one run only
  - `manifest.json` must be updated to the rebuilt file hash and size

Core schema rule:
- `trade_log.csv` and `bar_log_YYYYMMDD.csv` are retained as emitted.
- no post-processing may widen or rewrite the core CSV schema.
